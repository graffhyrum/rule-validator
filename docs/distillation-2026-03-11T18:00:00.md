## Patterns Worth Repeating

- **Pattern**: Fresh Eyes Audit
  **Condition**: Any session where the goal is "find bugs" or "audit the codebase"
  **Summary**: Read ALL source files and ALL test files in parallel batches before touching any code; trace execution from entry points; run `/simplify` after fixes.

- **Pattern**: Bead Batching by File Scope
  **Condition**: Multiple beads all modify the same file
  **Summary**: Read once, apply all changes in dependency order, run tests once — avoids merge conflicts and reduces iteration cycles.

- **Pattern**: Expert Review → Implementation Flow
  **Condition**: Session that produced non-trivial changes covering 3+ files or with suspected coverage gaps
  **Summary**: `bun fastvet` baseline → `/review:expert-review` (3 parallel agents) → implement high/medium → `bun fastvet` confirm green.
